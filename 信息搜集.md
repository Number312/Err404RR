# 信息搜集

## 1.指纹识别

步骤：

a. 看header  

b.看cookie 

c.跑whatweb 

d.dirsearch扫目录 

e.制造报错

### 1.1判断服务器类型（Nginx / Apache）

    大多抓包就能看出来

### 1.2判断语言（PHP / Java / ASP）

#### 1.2.1看响应头

 重点字段：

```http
Server: nginx/1.18.0
X-Powered-By: PHP/7.4.3
```

| 特征  | 说明  |
| --- | --- |

| `X-Powered-By: PHP` | PHP |
| ------------------- | --- |

| `X-Powered-By: ASP.NET` | ASP.NET（C#） |
| ----------------------- | ----------- |

| `Server: Apache + mod_php` | PHP |
| -------------------------- | --- |

| `Server: Microsoft-IIS` | ASP.NET |
| ----------------------- | ------- |

| `Set-Cookie: JSESSIONID` | Java |
| ------------------------ | ---- |

| `Set-Cookie: PHPSESSID` | PHP |
| ----------------------- | --- |

判断框架（Spring / Django / Laravel）

- 判断中间件（Tomcat / WebLogic）

- 判断组件（Shiro / Redis / Elasticsearch）

#### 1.2.2、看 Cookie（非常关键）

Cookie 直接暴露后端语言 

| Cookie 名            | 语言             |
| ------------------- | -------------- |
| `PHPSESSID`         | PHP            |
| `JSESSIONID`        | Java           |
| `ASP.NET_SessionId` | ASP.NET        |
| `csrftoken`（常见）     | Python（Django） |

#### 1.2.3 看URL的特征

虽然现在很多网站会隐藏，但依然有用：

| URL                        | 语言      |
| -------------------------- | ------- |
| `.php`                     | PHP     |
| `.jsp` / `.do` / `.action` | Java    |
| `.aspx`                    | ASP.NET |
| `.py`（少见）                  | Python  |

例如：

```
/login.jsp   → Java  
/index.php   → PHP  
/user.do     → Java（Spring）
```

#### 1.2.4 看报错信息

制造错误（比如传错参数），观察返回

### 常见报错特征：

PHP

`Warning: include(): Failed opening...  
Fatal error: Call to undefined function...`

Java

`java.lang.NullPointerException  
org.springframework...`

Python

`Traceback (most recent call last):`

 ASP.NET

`Server Error in '/' Application`

#### 1.2.6 、用工具自动识别

##### 1. whatweb

`whatweb http://target.com`

##### 2. wappalyzer（浏览器插件）

能看到：技术栈，第三方服务

#### 1.2.7、通过目录结构判断（进阶）

扫描目录：dirsearch  gobuster

`PHP典型目录：`

Laravel:

```
/app/  
/bootstrap/  
/vendor/  
/storage/  重点是后三个
/artisan  
.env
```

ThinkPHP

```
/application/  
/thinkphp/  
/runtime/  
/public/index.php
```

WordPress（CMS）

```
/wp-admin/  
/wp-content/  
/wp-includes/
```

`Java典型目录`

传统 Spring

```
/WEB-INF/  
/WEB-INF/web.xml  
/WEB-INF/lib/*.jar
```

Spring Boot

```
/actuator/  如果有返回 → 基本就是 Spring Boot
/error
```

`Python 典型结构`

Django：

```
/manage.py  
/db.sqlite3  
/static/  
/media/  
/templates/
```

Flask

```
/app.py  
/templates/  
/static/
```

`Node.js`

```
/node_modules/  
/package.json  关键
/app.js  
/server.js
```

Express 框架

```

/views/

```

`C#`

```
 /web.config
```
